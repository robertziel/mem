### Native Gate Sets per Hardware Modality

**What it is:**
Each physical qubit technology has a **native instruction set** — the gates the control electronics can execute directly at calibrated fidelities. Everything else is synthesized on top. Knowing the native set tells you where compilation pays off, where SWAPs will appear, and how many "free" parameters your variational circuit really has.

**The five major modalities:**
| Modality | Native 1-qubit | Native 2-qubit | Connectivity | Typical 2q fidelity |
|---|---|---|---|---|
| Superconducting (IBM, Rigetti) | `RZ(θ)`, `SX = √X`, `X` | `CZ` (Heron) or `ECR` | heavy-hex, nearest-neighbour | 99.7% |
| Superconducting (Google) | `PhasedXZ` | `fSim(θ,φ)`, `√iSWAP` | square lattice | 99.7% |
| Trapped ions (IonQ, Quantinuum) | `R(θ, φ)`, `RZ(θ)` | `MS(θ)`, `RZZ(θ)` | all-to-all (or zones) | 99.8–99.9% |
| Neutral atoms (QuEra, Atom) | `R(θ, φ)`, `RZ(θ)` | Rydberg `CZ`, multi-qubit `C_kZ` | reconfigurable | 99.5% |
| Photonic (PsiQuantum, Xanadu) | beam-splitter phases | KLM-style fusion gates | graph/cluster-state | probabilistic (boosted) |

**Details per modality:**

**Superconducting (IBM Heron/Condor):**
```
Basis: { RZ(θ),  SX,  X,  CZ }  — CZ replacing ECR in newest chips.
Arbitrary 1-qubit U = RZ(a) · SX · RZ(b) · SX · RZ(c)   (Euler-like)
```

**Google (Willow):**
```
1-qubit: PhasedXZ(x, z, a)  = Z(a) · X(x) · Z(z)
2-qubit: fSim(θ, φ)  or  √iSWAP with virtual Z
```

**Trapped ions (Quantinuum H2, IonQ Forte):**
```
1-qubit: R(θ, φ) = exp(-iθ(cos φ X + sin φ Y)/2),  RZ(θ) virtual
2-qubit: parametrized RZZ(θ) — θ generated by laser pulse length
         MS(θ) pairwise; global MS across multiple ions possible
```

**Neutral atoms (QuEra Aquila, Atom Computing):**
```
1-qubit: Raman-pulse R(θ, φ),  RZ(θ)
2-qubit: Rydberg blockade CZ (possibly C_kZ via blockade radius)
Geometry: movable atoms → reconfigurable connectivity per shot
```

**Photonic (KLM, measurement-based):**
```
Linear-optical interferometers + single-photon measurements
Fusion gates (Type-I / Type-II) — probabilistic, succeed with p = 1/2 boosted
Cluster-state resource; computation = measurement schedule
```

**Qiskit:**
```python
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit import QuantumCircuit, transpile

svc = QiskitRuntimeService()
backend = svc.backend('ibm_heron')
print(backend.target.operation_names)          # e.g., ['rz', 'sx', 'x', 'cz', 'measure', 'reset']

qc = QuantumCircuit(2); qc.h(0); qc.cx(0, 1)
native = transpile(qc, backend=backend, optimization_level=3)
print(native.count_ops())
```

**Decomposition overhead (non-native gates):**
| You wrote | Backend native | Cost per gate |
|---|---|---|
| CNOT | IonQ MS | 1 MS + 4 single-qubit |
| CNOT | Google fSim | 2 fSim + 4 single-qubit |
| Toffoli | any SC/ion | 6 CNOTs → 6× entangler cost |
| RZZ(θ) | IBM CZ | 2 CX + RZ (synthesis) |
| RZZ(θ) | IonQ | **native, zero overhead** |

**When to use this knowledge:**
- **Algorithm design**: express QAOA / VQE in the modality's native gates — 2–3× depth savings are common.
- **Benchmarking**: comparing algorithms across modalities needs normalized "logical gate" counts, not just CNOTs.
- **Resource estimation**: FTQC thresholds and magic-state costs depend on the native set's Clifford structure.

**Pitfalls:**
- **Paper circuits use CNOT**; real cost can be very different per modality.
- **All-to-all on ions is not free** — ion shuttling / global gates have their own costs and fidelities.
- **Photonic "gates" are probabilistic** — resource counts must include fusion success rate.
- **Native ≠ high fidelity**: Rydberg CZ is native on neutral atoms but often lower fidelity than SC CZ — always check the backend's calibration report.
- **Parametric gates get discretized**: even `RZZ(θ)` has finite calibration granularity (~mrad on modern ion traps).

**Rule of thumb:** Write the algorithm in terms of the backend's **actual** native gates — the transpiler will thank you, and you'll see 2–3× gate savings over the CNOT-normalized version.
