### Approximate Synthesis and T-Count Reduction

**What it is:** On fault-tolerant (FT) hardware the Clifford gates are cheap but the `T` gate (and other non-Clifford rotations) are exponentially expensive — each `T` consumes one magic state, which needs a full distillation factory. Compiling an arbitrary `Rz(theta)` into a Clifford+T sequence with `T`-count `O(log(1/epsilon))` and tolerated error `epsilon` is the central synthesis problem. Every T saved is real wall-clock saved on an FT device.

**Standard results:**

| Method                     | T-count for `Rz(theta)` to error `epsilon` | Notes                                           |
|----------------------------|--------------------------------------------|-------------------------------------------------|
| Solovay-Kitaev (SK)        | `O(log^{3.97}(1/epsilon))`                 | Generic, constructive, poor constants           |
| Ross-Selinger (gridsynth)  | `3 log2(1/epsilon) + O(log log 1/epsilon)` | Optimal leading constant, number-theoretic      |
| Repeat-until-success (RUS) | `~1.15 log2(1/epsilon)` expected T        | Probabilistic; shortest in expectation          |
| Programmable ancilla rotation (PAR) | Amortized < 1 T per rotation when many equal rotations | Batch technique                                 |

**Ross-Selinger / gridsynth:** Represents `Rz(theta)` as an element of `Z[omega]` (ring generated by eighth root of unity), solves a norm-one inequality over this ring via enumeration of lattice points — returns an exact Clifford+T word. Implementations: Haskell `gridsynth`, Python `pyqcgpu`, `pyzx`, and Qiskit's newer `SolovayKitaev`/`GenericBasisSynthesis` backends.

**RUS (repeat-until-success):** Produce a non-deterministic circuit with the desired rotation on success and a known correctable Pauli on failure. Measure flag qubits and retry on failure. Expected T-count ~1.15 log2(1/epsilon), worst-case unbounded — must bound retries in practice.

**Comparison in a worked example (Rz(pi/7) at epsilon = 1e-6):**

| Method              | T-count | Circuit depth | Deterministic? |
|---------------------|---------|---------------|----------------|
| Solovay-Kitaev      | ~250    | ~250          | Yes            |
| Ross-Selinger       | ~60     | ~60           | Yes            |
| Repeat-until-success| ~25 avg | variable      | No (retries)   |

**Qiskit — Solovay-Kitaev pass:**
```python
from qiskit import QuantumCircuit
from qiskit.transpiler.passes.synthesis import SolovayKitaev
from qiskit.transpiler import PassManager
from math import pi

qc = QuantumCircuit(1)
qc.rz(pi / 7, 0)   # arbitrary angle, not in Clifford+T natively

# Build or load a discretized basis lookup table once per run
sk = SolovayKitaev(recursion_degree=3)  # higher -> better epsilon, more gates
approx = PassManager([sk]).run(qc)
print(approx.count_ops())
# Typical output: {"h": N, "t": M, "tdg": K}  with M + K scaling log^3.97(1/epsilon)
```

For better scaling in production use `gridsynth` externally and import the gate sequence; or use Qiskit's `GenericBasisSynthesis` with an SK plugin that invokes Ross-Selinger internally.

**When to use:**
- Any compile aimed at an FT architecture (surface code, color code) — T-count is the dominant runtime.
- Batch synthesis: `Rz(theta_i)` for thousands of `theta_i` in a chemistry ansatz — time spent offline is recovered linearly per run.
- Variational circuits where the angles are frozen late — delay synthesis until after the last optimization iteration.

**Pitfalls:**
- SK's `log^{3.97}` prefactor means at `epsilon = 1e-6` you can easily get `hundreds` of T per Rz — always benchmark against gridsynth for production FT estimates.
- The "approximate" in approximate synthesis is a **coherent** error; combining N rotations each at error `epsilon` gives total error up to `N * epsilon` — budget accordingly.
- RUS's unbounded retries contaminate latency; enforce a cap and fall back to deterministic synthesis on failure.
- Qiskit's built-in `SolovayKitaev` uses SK, not Ross-Selinger — do not cite `3 log2(1/epsilon)` unless you verified the synthesizer.

**Example:** A chemistry VQE with 400 single-qubit rotations at epsilon=1e-4 — gridsynth gives ~40 T per rotation ~ 16k T total; SK gives 200 T per rotation ~ 80k T. On a 2us T cycle, that's 0.03s vs 0.16s per circuit — multiplied over shots, the difference is hours.

**Tips for practice:**
- Batch angles — many ansatz circuits share a small set of rotation angles; synthesize the distinct angles once and reuse.
- Align synthesis epsilon with the algorithm's tolerance — oversynthesizing wastes T-count, undersynthesizing contributes coherent error.
- For highly structured rotations (reflections, Hamiltonian simulation steps) consider specialized synthesis (e.g. qubitization, linear combination of unitaries) before resorting to generic gridsynth on each Rz.

**Rule of thumb:** T-count dominates fault-tolerant runtime — spend compile time (seconds of gridsynth per rotation is fine) because each avoided T is a factory-cycle's worth of real wall time saved on-device.
