### Stabilizer Formalism: Pauli Group and Stabilizer Codes

**What it is:** The algebraic framework underpinning nearly every practical QEC code (Shor, Steane, surface, color, LDPC). Encodes a k-qubit logical state in n physical qubits as the simultaneous +1 eigenspace of n − k commuting Pauli operators.

**The Pauli group P_n:**

P_n = {±1, ±i} × {I, X, Y, Z}^⊗n.

Every element is a tensor product of single-qubit Paulis with a phase. Key facts:
- |P_n| = 4·4^n elements.
- Any two Paulis either commute or anticommute.
- Weight wt(P) = number of non-identity tensor factors.

**Stabilizer group S:**

- S is an abelian subgroup of P_n not containing −I.
- Generated by n − k independent generators g_1, …, g_{n−k}.
- Each g_i is a multi-qubit Pauli of weight O(1) (local check).

**Code space:**

C(S) = { |ψ⟩ : g_i |ψ⟩ = +|ψ⟩ ∀ i }

Dimension dim C(S) = 2^k (encodes k logical qubits in n physical).

**Logical operators L:**

- Normalizer N(S) = {P ∈ P_n : P S P† = S}.
- Logical operators = N(S) / S (elements that commute with all stabilizers, modulo stabilizers themselves).
- For k logical qubits: 2k independent logical Paulis X̄_1, Z̄_1, …, X̄_k, Z̄_k.
- **Code distance** d = min weight of any non-trivial logical operator.

Notation: an [[n, k, d]] code.

**Error detection via syndromes:**

For an error E ∈ P_n:
- If E commutes with every g_i ⇒ syndrome = 0 (either no error, or logical error — undetectable).
- If E anticommutes with some subset ⇒ syndrome s(E) ∈ {0,1}^{n−k} reveals which generators flipped sign.
- Decoding: given syndrome s, find most-likely error E' with s(E') = s.

| Weight of E | Detected? | Correctable? |
|---|---|---|
| 0 | — | — |
| 1 to ⌊(d−1)/2⌋ | yes | yes |
| ⌊(d−1)/2⌋ to d−1 | yes | sometimes (degenerate) |
| ≥ d, logical | no | no |

**Why only Pauli errors?** Because the set {I, X, Y, Z} is a basis for 2×2 operators. Any single-qubit error (including arbitrary unitary or non-unitary CPTP map) can be decomposed as E = Σ c_P · P. If the code corrects X and Z on any qubit, it corrects any single-qubit error — this is the **discretization of errors theorem**.

**Syndrome extraction circuit (one generator g_i):**

```
ancilla: |0⟩ ─ H ── ● ── ● ── ● ── H ── M
data   : |ψ⟩ ─────── P_1 ─ P_2 ─ P_3 ──────
```
Measure ancilla in {|+⟩,|−⟩} basis → outcome = eigenvalue of g_i.

**Clifford group:** The normalizer of P_n. Generated by {H, S, CNOT}. Clifford gates map Paulis to Paulis — this is why Pauli frame tracking is cheap and why stabilizer codes admit transversal Clifford gates in many constructions.

**CSS codes (Calderbank–Shor–Steane):** A sub-class where every stabilizer is either all-X or all-Z. Built from two classical linear codes C_1 ⊇ C_2^⊥. X errors and Z errors decoded independently.

**Gottesman–Knill theorem:** Stabilizer circuits (Clifford + Pauli measurements + Pauli preparation) are classically simulable in polynomial time. Non-Cliffords (T, Toffoli) are required for quantum advantage — and for QEC, these need magic states.

**Stim code sketch for distance-3 repetition:**
```
R 0 1 2
X_ERROR(0.01) 0 1 2
M 0 1 2
DETECTOR rec[-1] rec[-2]
DETECTOR rec[-2] rec[-3]
```

**Rule of thumb:** If you can specify the stabilizer generators and logical operators in the Pauli group, you have a code; everything else (decoding, fault-tolerance, transversality) is corollaries from commutation relations.
