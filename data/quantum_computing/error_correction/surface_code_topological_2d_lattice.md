### Surface Code: 2D Topological Lattice QEC

**What it is:** The dominant fault-tolerant QEC code for nearest-neighbor-connected planar hardware (superconducting qubits, some neutral-atom systems). A stabilizer code on a 2D lattice of data qubits with ancillas at each plaquette. High threshold (~1%), local weight-4 checks, decodable with minimum-weight perfect matching in polynomial time. Google's Willow and IBM's upcoming Loon/Flamingo roadmaps target this code.

**Parameters:** [[d², 1, d]] for an unrotated patch (or ~d² / 2 data qubits for rotated variant). Distance d tunable — just make the patch bigger.

**Geometry:**
- Data qubits on vertices (or edges, depending on convention).
- Two types of plaquettes: X-type (measures ∏ X on 4 neighbors) and Z-type (measures ∏ Z on 4 neighbors).
- Alternating checkerboard.
- Each stabilizer generator has weight 4 (or 2 on boundary).

**Rotated surface code (used in practice):**
- d×d grid of data qubits + (d²−1)/2 X ancillas + (d²−1)/2 Z ancillas.
- Distance-3: 9 data + 8 ancillas = 17 qubits. Distance-5: 25+24=49. Distance-7: 49+48=97.

| d | Data | Ancilla | Total |
|---|---|---|---|
| 3 | 9 | 8 | 17 |
| 5 | 25 | 24 | 49 |
| 7 | 49 | 48 | 97 |
| 11 | 121 | 120 | 241 |
| 25 | 625 | 624 | 1249 |

**Stabilizers:**

X-plaquette: X_a X_b X_c X_d (for 4 neighbors a,b,c,d). Detects Z errors.
Z-plaquette: Z_a Z_b Z_c Z_d. Detects X errors.

All stabilizers commute because they either share 0 qubits or 2 qubits (even overlap ⇒ commute).

**Logical operators:**
- Logical X̄: string of X operators across the lattice from one smooth boundary to the other.
- Logical Z̄: string of Z operators across the lattice from one rough boundary to the other.
- Weight d — code distance.

**Error chains and decoding:**
- A local X error flips two adjacent Z-stabilizers → syndrome endpoints ("defects" or "anyons").
- A chain of X errors flips only the two endpoints (interior cancels).
- Decoder: pair up syndromes with shortest-weight matching → minimum-weight perfect matching (MWPM, Edmonds blossom, O(N³) worst case, fast heuristics: Fowler, PyMatching).
- Alternatives: Union-Find decoder (O(N α(N)), Delfosse–Nickerson), neural-net decoders (AlphaQubit from DeepMind, 2024).

**Threshold:**
- Phenomenological noise: ~1% per physical operation.
- Circuit-level noise (includes syndrome extraction errors): ~0.6–0.75% (Fowler et al.) for standard CZ-based.
- Google measured crossover to sub-threshold in 2024 (Willow, 105 qubits): d=3 → d=5 → d=7 showed logical error halving per increment → λ ≈ 2 per distance step.

**Logical error scaling (below threshold):**

P_L ≈ A · (p / p_th)^((d+1)/2)

"Below threshold" = increasing d suppresses logical error exponentially.

Willow (2024): p_L ≈ 0.14% per cycle at d=7; p_L halves when going from d=5 to d=7.

**Fault-tolerant gates:**
- Clifford gates via lattice surgery (merge/split patches). No transversal logic in a single patch — must deform.
- T gate: magic-state distillation + teleportation.
- Footprint: one logical qubit = patch, one T gate ≈ one distilled magic state.

**Overhead for fault-tolerant computation:**
- 1 logical qubit at d=25: ~1250 physical qubits, ~μs per round.
- A useful computation (Shor 2048-bit): ~10⁷ physical qubits × 10⁸ rounds (rough estimate from Gidney-Ekera 2021).

**Rule of thumb:** If you have a 2D planar chip and >1% two-qubit fidelity is out of reach, surface code is the default — budget ~1000 physical per logical at d~20 for a serious fault-tolerant application, and remember every T gate still costs you a magic-state factory.
