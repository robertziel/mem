### Remote async behavioral interview answers

### Blocked across time zones

- Document what you tried
- Explore one or two reasonable alternatives
- Leave a clear async update with the blocker, current findings, and a proposed next step
- Switch to another task while waiting when possible

### Making code understandable for a global async team

- Write PR descriptions that explain why, not just what
- Use meaningful commit messages
- Prefer clear names over clever ones
- Keep docs and ADRs current when decisions matter
- Let tests document expected behavior

### Talking about a production mistake

- Explain what happened clearly
- Show how you detected and mitigated it
- Focus on ownership, communication, and the blameless postmortem
- End with the process change that reduced the chance of repeating it

**Rule of thumb:** In remote-first interviews, show calm ownership, async clarity, and practical prevention steps.
